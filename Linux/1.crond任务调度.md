##  crond任务调度

描述：系统在某一时间反复执行特定任务

分类：系统工作（在系统运行的整个过程中的任务），用户工作（用户操作的程序）

语法：

- crontab [选项]

- 选项：

  -e：编辑crontab定时任务

  -l：查询crontab任务

  -r：删除当前用户所有的crontab任务

service crond restart 重启定时任务



###  示例

示例：执行crontab -e编辑定时任务

定时任务内容：`*/1**** ls -l /etc/ > /tmp/to.txt`

描述：在每小时第一分钟开始执行定时任务

参数：

| 占位符   | 含义               | 范围 |
| -------- | ------------------ | ---- |
| 第一个 * | 一小时中的第几分钟 | 0-59 |
| 第二个 * | 一天中第几个小时   | 0-23 |
| 第三个 * | 一个月中第几天     | 1-31 |
| 第四个 * | 一年中第几个月     | 1-12 |
| 第五个 * | 一周中第几个星期   | 0-7  |

每个*后可接数字格式

- 不接数字表示每分执行，每小时执行

- `,`：不连续的时间
- `-`：连续时间
- `/n`：每隔n时间执行一次

### 特定时间格式

| 时间         | 描述                                     |
| ------------ | ---------------------------------------- |
| 45 22 * * *  | 在每天22:45执行                          |
| 0 17 * * 1   | 每周一17:00执行                          |
| 0 5 1-15 * * | 每月1-15日5:00执行                       |
| */10 4 * * * | 每天4:00开始每隔十分钟执行，直到五点结束 |



### 使用crond调用shell脚本

my.sh内容：

```shell
date >> /home/mycal
cal >> /home/mycal
```

示例：执行crontab -e编辑定时任务

定时任务内容：`*/1**** /home/my.sh`

描述：每隔一分钟调用一次my.sh脚本
